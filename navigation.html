Welcome to navigation data editing. Here is where we edit ILS approach information, such as localizer and glide slope information as well as VORs, fixes, and airways.

Contributing

Getting started with navigation editing is extremely simple. The video above covers all of the information, but here are the steps anyway:

1. Create a GitHub account, by entering your details on the GitHub website. (Skip this step if you already have one.)

2. Direct message one of the airport editing admins on Slack your GitHub username. They'll invite you to the GitHub team. Accept that invite on the GitHub website or in your email. (If you're not a part of the Slack, scroll down to the Slack section to get more information on how to get invited.)

3. Install a text editor - default text editors such as TextEdit and NotePad++ work fine. If you want a better text editor, I recommend you install something from the internet.

4. Download the GitHub Desktop app.

5. Login to the GitHub Desktop app, then tap the "+ button on the top left corner, select the "Clone" tab, and select "Navigation". Save this to somewhere you'll remember later.

Committing Changes

  • Save the file you just edited.
  • Open GitHub Desktop and press the "Navigation" text on the left hand side.
  • You'll see your changes you made. Make sure the change(s) is ticked off.
  • On the bottom, enter the change description.
  • Press commit.
  • Sync with the repository by pressing the "Sync" button on the top right.
